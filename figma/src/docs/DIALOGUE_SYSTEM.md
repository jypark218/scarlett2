# 🎭 고도화된 대화 시스템 가이드

## 개요

모로 백작의 비밀 게임은 고도화된 자동 캐릭터 매칭 시스템을 사용하여 대화를 처리합니다.
이 시스템은 다양한 대화 형식을 자동으로 감지하고, 화자를 정확하게 매칭합니다.

## 📋 지원하는 대화 형식

### 1. 콜론 형식 (권장)
```
홈즈: 흥미롭군요, 왓슨. 이 사건은 생각보다 복잡해 보입니다.
왓슨: 그렇습니까? 저는 비교적 단순해 보이는데요.
```
**신뢰도: 100%** - 가장 명확하고 안정적인 형식

### 2. 인용부호 + 서술
```
홈즈가 말했다. "흥미롭군요, 왓슨."
그레그슨이 대답했다. "저희도 그렇게 생각합니다."
```
**신뢰도: 95%** - 서술이 포함되어 문맥이 풍부함

### 3. 역순 인용부호
```
"흥미롭군요." 홈즈가 말했다.
"어떻게 아셨죠?" 왓슨이 물었다.
```
**신뢰도: 95%** - 자연스러운 서술 형식

### 4. 순수 서술
```
홈즈가 고개를 끄덕였다.
왓슨은 놀란 표정을 지었다.
```
**신뢰도: 90%** - 행동 묘사, 내레이션으로 처리

### 5. 대괄호 형식
```
[홈즈]: 이것은 중요한 단서입니다.
[내레이션]: 방 안이 고요해졌다.
```
**신뢰도: 100%** - 명시적 화자 지정

## 🛠️ 개발자 도구 사용법

### 도구 열기
1. 개발 모드(`npm run dev`)로 실행
2. 게임 화면 우측 하단의 톱니바퀴(⚙️) 버튼 클릭

### 주요 기능

#### 1. 전체 검증
- **모든 노드 검증**: 전체 스토리의 대화를 검증
- **문제 노드 목록**: 에러나 경고가 있는 노드만 추출

#### 2. 개별 노드 테스트
```
노드 ID 입력 (예: start)
- [검증]: 해당 노드의 문제점 확인
- [파싱 테스트]: 대화가 어떻게 파싱되는지 확인
```

#### 3. 시스템 정보
- **지원 형식 확인**: 사용 가능한 대화 형식 목록
- **캐릭터 별칭**: 인식 가능한 캐릭터 호칭 목록

### 콘솔에서 직접 사용

브라우저 개발자 도구(F12) 콘솔에서:

```javascript
// 전체 검증
import { validateAllDialogues, printValidationReport } from './utils/dialogueValidator';
const report = validateAllDialogues();
printValidationReport(report);

// 특정 노드 검증
import { validateNode } from './utils/dialogueValidator';
validateNode('start');

// 파싱 테스트
import { testParseNode } from './data/enhancedCharacterData';
import { storyData } from './data/storyData';
testParseNode('start', storyData);
```

## 📊 검증 결과 이해하기

### 심각도 수준

#### ❌ 에러 (Error)
치명적인 문제로 즉시 수정 필요:
- 존재하지 않는 캐릭터 ID
- 빈 대화 내용

#### ⚠️ 경고 (Warning)
주의가 필요한 문제:
- 낮은 신뢰도 매칭 (< 70%)
- 선언된 캐릭터와 불일치

#### ℹ️ 정보 (Info)
개선 가능한 부분:
- 형식 문제 (인용부호 불일치 등)
- 신뢰도 보통 (70-80%)

### 제안 활용하기

각 문제에는 해결 제안이 포함됩니다:

```
⚠️ 낮은 신뢰도로 화자 감지됨 (65%)
텍스트: "이상하군요. 여기에 흔적이..."
감지된 화자: narrator
💡 제안: 가능한 화자: 셜록 홈즈 (점수: 8), 그레그슨 형사 (점수: 3)
```

**해결 방법:**
```
// 수정 전
"이상하군요. 여기에 흔적이..."

// 수정 후
홈즈: 이상하군요. 여기에 흔적이...
```

## 🔧 문제 해결

### 낮은 신뢰도 경고가 나올 때

1. **화자를 명시적으로 표시**
   ```
   // 수정 전
   "이상합니다."
   
   // 수정 후
   홈즈: 이상합니다.
   ```

2. **노드에 character 속성 추가**
   ```typescript
   {
     id: 'some_node',
     text: `"이상합니다."`,
     character: 'holmes',  // 추가
   }
   ```

### 캐릭터 불일치 경고가 나올 때

노드의 `character` 속성과 실제 대화 내용이 일치하는지 확인:

```typescript
// ❌ 불일치
{
  character: 'watson',
  text: `홈즈: 이것은 단서입니다.`
}

// ✅ 일치
{
  character: 'holmes',
  text: `홈즈: 이것은 단서입니다.`
}
```

## 📚 API 참고

### characterMatcher.ts
- `detectCharacter(text, context)`: 화자 자동 감지
- `extractDialogue(text)`: 순수 대사만 추출
- `parseDialogueLines(text, defaultChar, nodeId)`: 전체 파싱

### dialogueValidator.ts
- `validateAllDialogues()`: 전체 검증
- `validateNode(nodeId)`: 노드별 검증
- `getProblematicNodes()`: 문제 노드 목록

### enhancedCharacterData.ts
- `parseDialogueEnhanced(text, char, nodeId)`: 향상된 파싱
- `testParseNode(nodeId, data)`: 파싱 테스트
- `checkFormatSupport()`: 지원 형식 확인
- `checkCharacterAliases()`: 별칭 목록 확인
